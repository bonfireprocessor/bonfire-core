# Debugging and Legacy Flow

This page covers the legacy `tb_run.py` workflow and the simulated GDB server.

## Legacy `tb_run.py`

`tb_run.py` is the original test runner. It is still supported for debugging
and compatibility but the pytest-based workflow (`tests/` + `scripts/bonfire-core`)
is the recommended entry point for regular development.

### Module unit tests

```bash
python tb_run.py --ut_modules
```

Tests individual modules. Most testbenches are self-checking and raise an
assertion on error.

### Load/Store unit tests

```bash
python tb_run.py --ut_loadstore
```

Tests the load/store unit across different configurations and pipeline depths.

### Pipeline integration tests

```bash
python tb_run.py --pipeline
```

Tests the 3-stage pipeline backend alone and together with the fetch unit.

### Core integration test (`--hex`)

Runs the complete core in a testbench with 16 KiB RAM at address `0` and a
monitor port at `0x10000000`–`0x1fffffff`.

```bash
python tb_run.py --hex=code/build/core-tests/loadsave.hex
```

A write to `0x10000000` with value `1` signals success:

```text
Monitor write: @6830 10000000: 00000001 (1)
```

Pass `-v` to also log every executed instruction:

```bash
python tb_run.py -v --hex=code/build/core-tests/loadsave.hex
```

### VCD output

The `--vcd=<filename>` option enables writing a VCD waveform file (supported
for the core integration test only):

```bash
python tb_run.py --hex=code/build/core-tests/loadsave.hex --vcd=/tmp/loadsave.vcd
```

### SoC testbench (legacy mode)

The legacy MyHDL SoC run:

```bash
python tb_run.py --new_soc --hex=code/build/soc/sim/led.hex
```

This is superseded by:

```bash
pytest -vv tests/test_soc_myhdl.py
```

## Simulated GDB server

The simulated GDB server allows debugging RISC-V programs running in the
MyHDL simulation using `riscv64-unknown-elf-gdb`.

### Starting the server

Via the universal runner (recommended):

```bash
scripts/bonfire-core --gdbserver --port 1234
```

Or directly:

```bash
python -m gdbserver --hex code/build/debug-tests/endless.hex --port 1234
```

If `--port` is omitted, the server binds to the first free port in
`5500-5550`.

### Connecting with GDB

```bash
riscv64-unknown-elf-gdb code/build/debug-tests/endless.elf
```

Inside GDB:

```gdb
target remote 127.0.0.1:1234
```

## GHDL simulation (legacy FuseSoC command)

The older direct FuseSoC GHDL simulation command is still functional:

```bash
fusesoc --cores-root . run --target=sim bonfire-core --testfile=code/build/core-tests/loadsave.hex
```

This is mostly useful for debugging the VHDL conversion output.

## HEX file format

The testbench accepts a simple hex format: one 32-bit word per line in
hexadecimal:

```text
00a00513
10000417
ffc40413
000055b7
...
```

These files are generated by the `make all` build in `code/`.
